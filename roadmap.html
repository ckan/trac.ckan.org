A better way of deploying CKAN, including deploying multiple instances of CKAN on a single server, and handling upgrades, extensions, and themes. Likely based on a virtualenv in a debian package, and ansible. As far as possible we want there to be One True Way of installing CKAN, whether you're installing it on your laptop for development, on a continuous integration server to run the tests, deployment of a single instance, or deployment of multiple instances. As far as possible the install instructions for these different contexts should reuse the same ckanbuild components.
